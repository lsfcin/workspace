# skills
> Agent skills — provider-agnostic workflows invoked as slash commands or by instruction.

## Source of truth

`core/skills/<name>.md` is the **only** place to edit a skill. The files under `.opencode/skills/` and `.claude/skills/` are symlinks generated by `core/tools/sync-skills` — never edit them directly.

## Creating / editing a skill

1. Copy [`_template.md`](_template.md) → `<name>.md` (lowercase, kebab-case).
2. Fill the YAML frontmatter — `name` and `description` are required. `description` is what shows up in the agent's skill menu, so make it actionable.
3. Write the body: terse, prescriptive instructions the model follows when the skill fires.
4. Run `core/tools/sync-skills` to (re)generate both mirrors:
   - `.opencode/skills/<name>/SKILL.md` → `core/skills/<name>.md`  (opencode)
   - `.claude/skills/<name>/SKILL.md`  → `core/skills/<name>.md`  (Claude Code)
5. Verify with `core/tools/sync-skills --check` (exits non-zero if anything is stale/broken).

After sync, the skill is invocable from chat in both programs: `/skill-name [args]` or by natural-language trigger matching the `description`.

## Editing an existing skill

Just edit `core/skills/<name>.md` and re-run `core/tools/sync-skills`. No mirror edits needed — they're symlinks.

## Case-sensitivity warning

Linux is **case-sensitive**. The source dir is `core` (lowercase). Never write `Core/...` in a symlink target — that was the root cause of the `Failed to parse skill` error on startup (dangling symlinks).

## Excluded from mirroring

`_template.md`, `*.original.md` (caveman-compress backups), and `CONTEXT.md` are not skills — `sync-skills` skips them.

## Global skills (outside the workspace)

The `caveman*` / `cavecrew` skills live in `~/.agents/skills/` and are mirrored via `~/.claude/skills/`. They are installed system-wide and not synced by `core/tools/sync-skills`.
